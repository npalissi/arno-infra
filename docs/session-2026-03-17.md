## Problèmes connus / non résolus

1. **Tesseract.js ne supporte pas les PDF** — seulement images (.jpg, .png, .webp, .bmp, .tiff). Pour les factures PDF, il faudra soit convertir en image (pdfjs-dist) soit passer à un modèle vision IA.

2. **Parser Auto1 incomplet** — ne récupère que `galleryImages`. Il manque `highlightItems` (détails/état) et `damageItems` (dégâts). Voir `src/lib/auto1/client.ts:147-155`.

3. **Pas de deploy Vercel** — l'app n'est pas encore déployée. Vercel CLI pas installé.

4. **OCR factures non testé** — le script est prêt mais n'a pas encore été testé sur de vraies factures. À tester pour décider si OCR seul suffit ou s'il faut l'IA.

5. **Hydration mismatch** — le `mounted` guard masque tout le contenu au SSR (retourne `null`). Acceptable pour une app interne mais pas idéal pour le SEO. Solution alternative : wrapper uniquement les composants Base UI problématiques.

6. **Type casts Supabase** — `as unknown as` partout car `select('*')` retourne `{}`. Pas critique mais bruyant.

7. **Geist font** — installé (`geist` v1.7.0) mais le CLAUDE.md mentionne encore Inter par endroits. Vérifier la cohérence.
